✦ What's New
Unified Applications Tab
Install and Uninstall are now combined in a single Applications page with a segmented mode bar. Switch between Install and Uninstall instantly without navigating away. Category sidebar auto-hides in Uninstall mode.
Update All Apps button
A new ↑ Update All Apps button in the mode bar launches an external PowerShell window running winget upgrade --all --accept-source-agreements --accept-package-agreements -h — completely non-blocking. WinTooler stays fully responsive while updates run.
Background App ID Validator
On first visit to the Applications tab, a background pool of 8 concurrent workers runs winget search --exact for all 111 app IDs. Each badge updates live: ✓ OK or ⚠ Not found. Runs once per session, never blocks the UI.
CLI Progress Bar — Install & Uninstall
Both install and uninstall operations now show a live ASCII progress bar in the CLI window for every app, including the result status inline: Installed, Already present, or Failed (0xHHHHHHHH).
Compact App List
Removed the macOS-style icon tiles from every app row. Each row now shows: app name (bold) + description and package ID on one compact line. More apps visible at once with less scrolling.
In-App Language Toggle (EN / ES)
Switch between English and Spanish instantly using the pill toggle in the sidebar footer. All labels, titles, buttons, and status text update immediately — no restart required.
🔧 Bug Fixes
Uninstall via winget with Progress Bar
Uninstall now runs winget uninstall --id with full exit code reporting and an ASCII progress bar matching the install flow. Shows Removed or Failed (0xHHHHHHHH) per app. List refreshes automatically after completion.
Windows Update Scan — Removed --include-unknown
The --include-unknown flag was removed from the Windows Update tab scan. It produced noisy output. The scan now runs a clean winget upgrade with no extra flags.
Update All Apps — Silent Mode (-h)
The external upgrade window now runs winget upgrade --all --accept-source-agreements --accept-package-agreements -h. The -h flag enables silent/headless mode — no interactive prompts per package.
Install Cascade — 6 Retry Attempts
App installs now attempt up to 6 strategies before reporting failure — covering scope conflicts, CDN HTTP 404s, and hash mismatches automatically.
#ScopeHash BypassCovers
1PreferredNoNormal install
2OppositeNoScope conflict
3NoneNoScope incompatible
4PreferredYesHash mismatch / CDN 404
5OppositeYesHash + scope conflict
6NoneYesBroadest fallback
App IDs Corrected
Three incorrect winget package IDs fixed from the previous build:
OpenWhisperSystems.Signal → Signal.Signal
Insomnia.Insomnia → Kong.Insomnia
REALiX.HWiNFO → REALiX.HWiNFO64
Startup No Longer Blocks on winget Scan
The blocking winget upgrade --source winget scan that ran before the GUI opened has been removed. The app now opens instantly. Updates are scanned on demand via the Update All Apps button.
Restore Point — 24hr Warning Suppressed
The verbose Windows WARNING about the 1440-minute cooldown is no longer shown. WinTooler now pre-sets SystemRestorePointCreationFrequency = 0 before creating the restore point and restores the original value afterward. Silent and clean on every launch.
Language Toggle Crash Fixed
Inline if (…) {…} else {…} expressions used as function arguments inside WPF event handlers cause a PowerShell parser crash. All color values are now pre-computed into variables before being passed to brush helpers.
Uninstall Search Freeze Fixed
Typing in the search box while on the Uninstall tab no longer freezes the UI. The winget call (Fetch-UninstallData) is now cached and separated from the filter function (Build-UninstallList) which only operates on already-fetched data.

⚠️ Known Limitations
winget CDN Hash Mismatches
Some apps (Signal, Insomnia, HWiNFO64) occasionally fail due to publisher CDN updates not yet merged in the winget manifest. The 6-attempt install cascade with --ignore-security-hash handles these automatically.
DaVinci Resolve Install
DaVinci Resolve requires a Blackmagic Design account to download. The winget installer will open their website if the direct download is blocked.
Tweaks require Reboot
Some tweaks (power plan, hibernation, service changes) take full effect only after a system restart.
