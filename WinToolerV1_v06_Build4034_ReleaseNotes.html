Complete Bug Fix Log
8 bugs total
BUG-01
Startup screen: Launch button clipped at the bottom
v0.5 BETA
Fixed in v0.6 ▼ Details

What happened: The Launch WinToolerV1 button was partially or fully hidden at the bottom of the startup window. Because ResizeMode="NoResize" was set, users could not resize to reach it.

Root cause: The button was injected via Add_ContentRendered after the window had already been measured and sized. WPF does not re-expand a fixed-height window for runtime content additions.

Fix: Button moved into XAML Grid at declaration time. SizeToContent="Height" added so the window auto-fits at any DPI or scale setting.

BUG-02
Theme toggle icon showing literal &#x2600; text
v0.5 BETA
Feature Removed ▼ Details

What happened: After clicking the Dark/Light toggle, the button label changed from the expected sun or moon symbol to the raw XML entity string displayed as plain text.

Root cause: XML entities are decoded by the XAML parser at load time only. Assigning the same string via PowerShell passes it as a literal — no entity decoding occurs.

# v0.5 — broken: literal 8-char string $ctrl["ThemeIcon"].Text = if ($dark) { "&#x2600;" } else { "&#x263D;" } # v0.6 — fixed: actual Unicode codepoint $ctrl["ThemeIcon"].Text = if ($dark) { [char]0x2600 } else { [char]0x263D }

Resolution: Dark mode and theme toggle removed entirely in v0.6. Light mode only.

BUG-03
Light mode incomplete — most panels staying dark
v0.5 BETA
Fixed in v0.6 ▼ Details

What happened: Switching to light mode changed only the window background and sidebar. All tab content — cards, toolbars, search boxes, output consoles, text labels — remained dark.

Root cause (two layers): Dynamic panels were built in PowerShell with hardcoded dark FromRgb values. XAML structural surfaces had hardcoded dark Background values with no x:Name, so Apply-Theme could not reference them.

Fix: All XAML dark values replaced at source. Dynamic panel builders updated to read from $script:T token palette. Apply-Theme extended to repaint all panels in-place. Dark mode then removed permanently.

BUG-04
ISO Downloader frozen indefinitely on "Fetching..."
v0.5 BETA
Feature Removed ▼ Details

What happened: Clicking Get Download Link showed "Fetching..." indefinitely. The spinner ran forever regardless of edition, architecture, or language selected.

Root cause: Microsoft's Software Download API requires an active browser session cookie established by first visiting the download page. A cold script call with no session step returns an empty or error response. The polling timer waited forever for a job that had already silently failed.

Resolution: ISO Downloader tab removed from this release. Will be re-evaluated in a future build once a stable cross-region method is confirmed.

BUG-05
Active nav button not reflecting light mode
v0.5 / v0.6 BETA
Fixed in v0.6 ▼ Details

What happened: Even after switching to light mode, the active navigation button stayed dark blue — visually wrong against the white sidebar.

Root cause: NavBtnActive was a XAML Style resource with hardcoded dark colors. WPF style precedence meant runtime property assignments from Apply-Theme could not override style-set values.

Fix: switchPage no longer assigns a style object. It now directly sets Background, Foreground, and FontWeight using $script:T tokens at the moment of navigation.

BUG-06
GUI crash: Missing expression after comma
v0.6 BETA
Fixed in Build 4.034 ▼ Details

Error:

At gui.ps1:1185 char:48 + "AboutTitle","AboutSub","AboutVersion", + ~ Missing expression after ','.

Root cause: Trailing comma after the last element of the $names array. PowerShell requires the next element to follow a comma — a trailing comma at end of array is a syntax error that prevents any code in the file from running.

Fix: Trailing comma removed. "DNSCardPanel","DNSOutput","DNSStatusTxt" added as the actual last elements of the array.

BUG-07
GUI crash: RowDefinitions property already set
v0.6 BETA
Fixed in Build 4.034 ▼ Details

Error:

Exception calling "Load": "'RowDefinitions' property has already been set on 'Grid'."

Root cause: During the Fluent Design XAML refactor, the new Window.Resources replacement block ended with <Grid><Grid.RowDefinitions>...</Grid.RowDefinitions> to begin the layout — but the original code already had that same Grid and its RowDefinitions block immediately after </Window.Resources>. WPF aborted loading because the same property was declared twice on the same element.

Fix: Duplicate <Grid.RowDefinitions> block removed. One declaration remains.

BUG-08
GUI crash: Unknown member CharacterSpacing
v0.6 BETA
Fixed in Build 4.034 ▼ Details

Error:

Exception calling "Load": "Cannot set unknown member 'System.Windows.Controls.TextBlock.CharacterSpacing'."

Root cause: CharacterSpacing is a WinUI 3 / UWP property that does not exist on System.Windows.Controls.TextBlock in WPF (.NET Framework). WPF throws immediately when it encounters an unknown property on a known type — it does not silently ignore unknown attributes.

Fix: CharacterSpacing="80" removed from all three nav section label TextBlock elements.
