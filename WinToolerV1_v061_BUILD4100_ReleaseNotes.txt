OVERVIEW
--------
WinToolerV1 is a lightweight Windows system utility built with PowerShell + WPF.
Install 111 curated apps via winget, apply performance and privacy tweaks, manage
services, and run system repair tools — all from one clean bilingual GUI.

================================================================================
  WHAT'S NEW IN BUILD 4100
================================================================================

[NEW] Unified Applications Tab
  Install and Uninstall are now combined in a single Applications page with a
  segmented mode bar (Install | Uninstall). Category sidebar auto-hides in
  Uninstall mode. Switch modes instantly without navigating.

[NEW] Update All Apps button
  A new "Update All Apps" button in the mode bar launches an external PowerShell
  window running:
    winget upgrade --all --accept-source-agreements --accept-package-agreements -h
  WinTooler stays fully responsive while updates run. Winget path is validated
  before launching — shows a warning if winget is not installed.

[NEW] Background App ID Validator
  On the first visit to the Applications tab, a background worker pool (8
  concurrent jobs) runs "winget search --id --exact" for all 111 app IDs.
  Each app badge updates in real time:
    [checkmark] OK         — ID found in winget
    [warning]   Not found  — ID not found in winget source
  Runs once per session. Does not block the UI.

[NEW] In-App Language Toggle (EN / ES)
  Switch between English and Spanish instantly using the pill toggle in the
  sidebar footer. All labels, titles, buttons, and status text update
  immediately — no restart required.

[NEW] CLI Progress Bar — Install & Uninstall
  Both install and uninstall operations now show a live ASCII progress bar
  in the CLI window for every app, with the result status shown inline:
    [############################] 100%  App Name  >> Installed
    [############################]  ERR  App Name  >> Failed (0xHHHHHHHH)

[NEW] Compact App List
  Removed the macOS-style icon tiles from every app row. Each row now shows:
  app name (bold) + description and package ID on one compact line. More apps
  visible at once without excessive scrolling.


================================================================================
  BUG FIXES
================================================================================

[FIX] Uninstall via winget with Progress Bar
  Uninstall now runs "winget uninstall --id" with full exit code reporting
  and an ASCII progress bar matching the install flow. The list refreshes
  automatically after all selected apps are removed.

[FIX] Windows Update Scan — Removed --include-unknown
  The --include-unknown flag was removed from the Windows Update tab scan.
  It produced noisy, misleading output. The scan now runs a clean
  "winget upgrade" with no extra flags.

[FIX] Update All Apps — Silent Mode (-h)
  The external upgrade window now runs:
    winget upgrade --all --accept-source-agreements --accept-package-agreements -h
  The -h flag enables silent/headless mode — no interactive prompts per package.

[FIX] App Install — 6-Attempt Retry Cascade
  App installs now try up to 6 strategies before reporting failure. Each
  attempt varies scope and hash-bypass flags to cover the most common failure
  modes automatically:

    Attempt 1: preferred scope                         (normal install)
    Attempt 2: opposite scope                          (scope conflict)
    Attempt 3: no scope flag                           (scope incompatible)
    Attempt 4: preferred scope + --ignore-security-hash (hash mismatch / 404)
    Attempt 5: opposite scope  + --ignore-security-hash (hash + scope conflict)
    Attempt 6: no scope        + --ignore-security-hash (broadest fallback)

  Error messages now display the hex exit code when all 6 attempts fail, making
  diagnosis much easier.

[FIX] App IDs Corrected
  Three package IDs were wrong in the previous build:
    OpenWhisperSystems.Signal  -->  Signal.Signal
    Insomnia.Insomnia          -->  Kong.Insomnia
    REALiX.HWiNFO              -->  REALiX.HWiNFO64

[FIX] Startup No Longer Blocks on winget Scan
  The blocking "winget upgrade --source winget" scan that ran in the CLI
  bootstrap window before the GUI opened has been removed. The app now opens
  instantly. Updates are checked on demand via the Update All Apps button.

[FIX] Restore Point — 24-Hour Warning Suppressed
  The Windows WARNING about the 1440-minute cooldown is no longer printed.
  WinTooler now temporarily sets SystemRestorePointCreationFrequency = 0 before
  creating the restore point and restores the original value afterward.
  Registry key: HKLM\Software\Microsoft\Windows NT\CurrentVersion\SystemRestore

[FIX] PSWindowsUpdate Row Removed from Boot
  The "PSWindowsUpdate — Not required (using winget)" row no longer appears in
  the CLI dependency check at startup.

[FIX] Language Toggle Crash
  Using inline if-expressions as function arguments inside WPF event handlers
  causes a PowerShell parser exception. All color values are now pre-computed
  into local variables before being passed to brush helper functions.

[FIX] Uninstall Search Freeze
  Typing in the search box while on the Uninstall tab no longer freezes the UI.
  The winget list call (Fetch-UninstallData) is cached on first load; the filter
  function (Build-UninstallList) only operates on already-fetched in-memory data.

[FIX] PowerShell 5.1 Compatibility
  Replaced ?. null-conditional operator (PS7+ only) with PS5.1-compatible
  conditional assignment. WinTooler runs on Windows PowerShell 5.1 as shipped
  with Windows 10 / 11 — no PowerShell 7 installation required.

================================================================================
  REQUIREMENTS
================================================================================

  OS:          Windows 10 (Build 19041 / 20H1 or newer) or Windows 11
  PowerShell:  5.1 (built-in — PowerShell 7 is NOT required)
  winget:      App Installer from Microsoft Store (required for Apps tab)
  Privileges:  Administrator (required for tweaks, services, repairs)


================================================================================
  HOW TO RUN
================================================================================

  1. Extract WinToolerV1_v061_BETA_BUILD4100.zip to any folder
  2. Right-click Launch.bat → Run as administrator
  3. CLI bootstrap window runs dependency checks and creates a restore point
  4. GUI opens automatically


================================================================================
  KNOWN LIMITATIONS
================================================================================

  - winget CDN hash mismatches: Signal, Insomnia, and HWiNFO64 occasionally
    fail due to publisher CDN updates not yet merged in the winget manifest.
    The 6-attempt cascade with --ignore-security-hash handles this automatically.

  - DaVinci Resolve: Requires a Blackmagic Design account to download.
    winget may open their website if direct download is blocked.

  - Some tweaks (power plan, hibernation, service changes) require a full
    system restart to take effect.

  - App ID validator runs winget queries in background — may take 2-3 minutes
    to complete validation of all 111 IDs depending on network speed.
